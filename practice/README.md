# 199명 홀수인원을 처리 가능한가?

- 1번 타이밍에 99명의 인원이 식사 가능.
- 2번 타이밍에 99명의 인원이 식사 가능
- 번호를 매겨보면 0 ~ 198(index 기준), 1 ~ 199 (number 기준).
- number 기준으로 2번, 4번, 6번 ... 198번 즉 99명에게 우선적으로 식사 공급.
- 1번 3번 5번 ... 199번이 그 다음 식사를 하게 될텐데, 이때 1번과 199번이 경쟁을 하게됨.
- 1번이 이긴다고 가정, 1번은 자신의 포크와 199번 포크로 식사를 함.
- 3번 타이밍에서 짝수인원들과 경쟁을 하게 되는데 199번은 198번과 경쟁하게됨.
- 이때 198번이 이길 경우, 199번이 사망하게 됨.
- 그러면 경계인원에서 적게 먹은 친구를 추적해서 우선권을 줘야함.
- 홀수일 경우에만 추적해야함
- 제일 처음에는 짝수인원을 먹이고, 그다음 홀수인원을 먹이되,
- 1번과 199번 중 누가 못먹는지를 체크한다.
- 그 다음 먹을때에는 누구를 굶길건지에 대해서 락을 건다. 
- 199번을 굶기게 할 경우, 다음 싸이클에서 198번과 경쟁을 하게됨. 
- 그렇게 되면 198번을 굶겨야되고, 197번과 경쟁하게됨. 
- 즉 한번 굶는 방향을 정하면, 고정되게 굶기시작함.

- 그럼 여기서 어떻게 구현할 것인가?
- 모두에게 
- must_do_not_starve 에 굶을 인원 등록하고, mutex_lock을 걸어버림.
- 인덱스를 하나씩 늘려가며 처리.
- 그럼 어떻게 짝수인원들 부터 밥을 먹이지?
- 하나하나 뮤텍스를 거는 것은 비효율로 보임.
